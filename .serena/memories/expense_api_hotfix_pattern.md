# 経費API ホットフィックスパターン

## 問題パターン
型定義の重複によりコンパイル時にエラーが検出されず、実行時エラーが発生

## 修正手順（段階的アプローチ）
### Phase 1: 緊急対応（15分）
1. エラー箇所のプロパティ名を正しいものに修正
2. 最小限の変更で画面を復旧

### Phase 2: 根本修正（20分）
1. 重複した型定義を削除
2. 正式な型定義をインポート
3. TypeScriptビルドで確認

### Phase 3: 予防措置（25分）
1. バグ再現テストを追加
2. 同様パターンの検索
3. ドキュメント更新

## 教訓
- 緊急時は最小限の修正で復旧を優先
- その後で根本的な問題を解決
- 予防措置で再発を防ぐ

## 確認コマンド
```bash
# 型チェック
npx tsc --noEmit

# テスト実行
npm test -- expense

# リント
npm run lint
```
